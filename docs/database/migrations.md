# Database Migrations

## Workflow

### 1. Modify the schema
Edit `app/shared/lib/db/schema.ts`.

### 2. Generate migration
```sh
npx drizzle-kit generate
```
Creates `db/migrations/NNNN_<description>.sql`.

### 3. Apply locally
```sh
wrangler d1 execute DB --local --file db/migrations/NNNN_<description>.sql
```

### 4. Apply to production
```sh
wrangler d1 execute DB --file db/migrations/NNNN_<description>.sql
```

### 5. Commit
Commit both `schema.ts` and `db/migrations/NNNN_*.sql` in the same PR.

## Rules

- **Never edit generated SQL files.** If a migration has an error, create a new migration to fix it.
- **Never delete migration files.** They are the canonical history.
- Migrations run in order — always test locally before production.
- The CI pipeline applies pending migrations before running integration tests.

## Rollback

D1 doesn't support transactional DDL rollback. Strategy:

1. Write a reverse migration manually (e.g., `DROP COLUMN` or restore table)
2. Test the reverse migration locally
3. Apply to production

For destructive changes (dropping columns, renaming tables), keep the old data:
- Add the new column/table first
- Backfill data
- Deploy code that uses the new column/table
- In a separate release, drop the old column/table

## Initial Migration

```sql
-- db/migrations/0000_initial.sql
-- This file is generated. Do not edit.

CREATE TABLE conversations (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  title TEXT NOT NULL DEFAULT '',
  model TEXT NOT NULL,
  system_prompt TEXT NOT NULL DEFAULT '',
  temperature REAL,
  max_tokens INTEGER NOT NULL,
  message_count INTEGER NOT NULL DEFAULT 0,
  total_input_tokens INTEGER NOT NULL DEFAULT 0,
  total_output_tokens INTEGER NOT NULL DEFAULT 0,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL,
  deleted_at INTEGER
);

-- (full schema generated by drizzle-kit generate)
```

## Checking Applied Migrations

```sh
# View schema in local D1
wrangler d1 execute DB --local --command ".schema"

# View schema in production D1
wrangler d1 execute DB --command ".schema"
```
